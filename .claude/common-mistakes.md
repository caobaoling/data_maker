# 常见错误记录

## 前端API路径错误

### 问题描述
在创建新的前端API调用时，经常忘记添加 `/api` 前缀，导致404错误。

### 错误示例
```javascript
// ❌ 错误写法
export function addPreContract(data) {
  return request({
    url: '/teacher/add_pre_contract',  // 缺少 /api 前缀
    method: 'post',
    data
  })
}
```

### 正确写法
```javascript
// ✅ 正确写法
export function addPreContract(data) {
  return request({
    url: '/api/teacher/add_pre_contract',  // 必须包含 /api 前缀
    method: 'post',
    data
  })
}
```

### 原因分析
1. 前端使用Vite代理配置，所有 `/api` 开头的请求会被代理到后端
2. 后端Flask应用注册蓝图时使用了 `url_prefix='/api/xxx'`
3. 完整的请求路径应该是：`/api/{蓝图名称}/{路由路径}`

### 检查清单
创建新的API接口时，务必检查：
- [ ] 前端API文件中的URL是否以 `/api/` 开头
- [ ] 后端蓝图注册时的 `url_prefix` 是否正确
- [ ] 前端和后端的路径是否匹配

### 相关文件
- 前端API配置：`frontend/src/api/*.js`
- Vite代理配置：`frontend/vite.config.js`
- 后端蓝图注册：`backend/app.py`

### 发生日期
- 2026-03-06: teacher.js API路径错误

## ⚠️ 重大原则性错误：擅自修改非需求相关代码

### 🔴 错误描述
在2026年1月某次需求中，用户报告部分页面404错误。在诊断问题时，发现某些API文件缺少 `/api` 前缀。**但错误的是**：在未经需求要求的情况下，擅自修改了 `elf.js`、`aiTeacher.js`、`redis.js`、`picturebook.js` 等多个文件的API路径，而这些文件可能本身就是按照某种特定方式配置的，或者根本不是问题所在。

### 🚫 核心问题
**违反了最基本的开发原则：未经需求要求，不得修改现有代码！**

### ❌ 错误行为模式
1. 看到代码"不一致"就擅自"统一"
2. 没有先验证这些代码是否真的有问题
3. 没有询问用户就批量修改多个文件
4. 自作主张地"优化"和"修复"

### ✅ 正确的处理方式应该是
1. **先诊断问题**：仔细分析404错误的真正原因
2. **只改需求相关的部分**：如果用户只报告特定页面404，只修复那个页面相关的代码
3. **询问用户**：发现可能的问题时，先询问用户是否需要修改
4. **参考已有代码**：新增功能时，参考项目中已经正常工作的代码（如 `appoint.js`、`teacher.js`、`user.js`）
5. **最小化修改**：遵循"最小修改原则"，只改必须改的

### 📋 绝对禁止清单
- ❌ 未经需求要求，不得修改API调用路径
- ❌ 未经需求要求，不得修改已有的配置文件
- ❌ 未经需求要求，不得"统一"代码风格
- ❌ 未经需求要求，不得"优化"现有代码
- ❌ 看到"可能的问题"时，不得擅自批量修改

### 🎯 正确的工作原则
```
用户明确需求 → 分析相关代码 → 只修改需求涉及的部分 → 其他代码保持不变
```

### 📝 用户原话（需铭记）
> "切记，以后添加需求，非需求设计的代码不要进行修改，尤其调用api的接口路径。不允许再次犯错，记到你的小本本里"

### 发生日期
- **2026年1月（具体日期未知）**: 擅自修改了4个API文件的路径，违反了"只改需求相关代码"的原则

### ⚡ 永久性警告
**此错误性质严重，必须永久警惕！每次修改代码前，必须自问三遍：**
1. 这是用户明确要求修改的吗？
2. 这个修改是需求必须的吗？
3. 我是否在擅自"优化"非需求代码？

# cocos 工具集说明文档

> 本模块用于课后推荐题接口的数据拉取、整理与校验，围绕同一份接口响应数据（`json.json`）形成完整的工作流。

---

## 目录结构

```
cocos/
├── fetch_api.py                    # 拉取接口数据，输出 json.json
├── extract_json_to_dataframe.py    # 解析 json.json，生成整理后的 Excel 报表
├── verify_mastery.py               # 校验接口返回数据是否符合掌握度起点规则
├── 题型定义映射关系.xlsx             # 题型编码 → 中文名称（verify_mastery.py 使用）
├── 需求的对应关系.xlsx               # 知识类型+能力标签+掌握度起点 → 允许题型（verify_mastery.py 使用）
├── GC_L0~L2_有效学习目标.xlsx        # 课型+Level → 有效知识点类型+能力项（verify_mastery.py 使用）
└── README.md                       # 本文档
```

---

## 推荐使用流程

```
step 1: 运行 fetch_api.py                  → 获取接口的返回内容，生成 json.json
step 2: 运行 extract_json_to_dataframe.py  → 将接口返回的 json 生成整理结果_时间戳.xlsx（人工核查用）
step 3: 运行 verify_mastery.py             → 控制台输出校验报告（手动执行接口，手动修改数据库，自动验证结果）
```

---

## 脚本详细说明

### 1. `fetch_api.py` — 拉取接口数据

**作用：** 根据 `appoint_id` 查询数据库获取 `material_id`，再调用课后推荐题接口，将原始响应保存为 `json.json`。

**使用方式：**
```bash
python cocos/fetch_api.py <appoint_id>
```

**输出：** `cocos/json.json`

---

### 2. `extract_json_to_dataframe.py` — 生成整理报表

**作用：** 读取 `json.json`，提取题目的知识点信息，输出带格式的 Excel 表格，便于人工核查接口数据结构。

**使用方式：**
```bash
python cocos/extract_json_to_dataframe.py
```

**输出：** `整理结果_YYYYMMDD_HHMMSS.xlsx`

---

### 3. `verify_mastery.py` — 掌握度起点校验（主校验脚本）

#### 3.1 功能概述

对照数据库中用户的知识点掌握度，验证课后推荐题接口是否按需求规则返回了正确的题型和题数。支持可选的课程范围过滤：当指定课型和 Level 时，仅校验该节课涉及的知识点组合。

#### 3.2 使用方式

```bash
# 模式一：校验全部知识点（不限课程范围）
python cocos/verify_mastery.py <appoint_id>

# 模式二：仅校验指定课型和 Level 涉及的知识点
python cocos/verify_mastery.py <appoint_id> <课型> <Level>
```

**参数说明：**

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `appoint_id` | 是 | 预约 ID | `539673674` |
| `课型` | 否 | 与 `GC_L0~L2_有效学习目标.xlsx` 中"课型"列完全一致 | `词汇句型课` |
| `Level` | 否（与课型同时提供） | 与 xlsx 中"Level"列完全一致 | `Level 0` |

**可选课型值：** `词汇句型课`、`对话课`、`阅读课`、`字母课`、`自拼课`、`嘉年华`

**可选 Level 值：** `Level 0`、`Level 1`、`Level 2`

**示例：**
```bash
python cocos/verify_mastery.py 539673674 词汇句型课 "Level 0"
python cocos/verify_mastery.py 539673674
```

#### 3.5 校验规则详解

校验的核心单位是**分组**：将同一（知识类型中文, 能力标签中文）下的所有知识点归为一组，整组统一校验。

**分组后按知识点个数分两条路径：**

---

##### 路径 A：知识点个数 = 1

每个有效掌握度层级（mastery_min ≤ lv ≤ 3）出 1 道题，合计期望 N 道（N = 有效层级数）。

**有效层级**：在 `需求的对应关系.xlsx` 中该（知识类型, 能力标签, lv）组合有对应题型（非 `--`）。

**题数上限：** 全局 20 道上限，当前分组的期望题数 = min(原始期望, 剩余配额)。

**校验内容：**
- 实际返回题数是否等于期望题数
- 实际返回题型是否在该组合所有层级允许的题型集合内

---

##### 路径 B：知识点个数 > 1（分层校验）

按层从低到高（mastery_min → mastery=3）逐层校验，每层过滤掉低层已使用的题型：

**mastery=1 层：**
- 参与出题的知识点：该组中 mastery ≤ 1 的知识点
- 规则：从该层允许题型中随机选 1 种，每个知识点各出 1 道
- 校验：同层题型唯一、题型合法、题数正确（受20道上限约束）

**mastery=2 层：**
- 参与出题的知识点：该组中 mastery ≤ 2 的知识点
- 规则：过滤掉 mastery=1 层已用题型后，随机选 1 种，每个知识点各出 1 道
- 若过滤后无可用题型：该层不出题，不报错
- 校验：同层题型唯一、题型合法、不复用低层题型（有其他可选时）

**mastery=3 层（连线题优先规则）：**

| 知识点数 N | 规则允许连线题 | 期望出题 |
|-----------|--------------|---------|
| N = 1 | 有非连线题优先 | 1 道（非连线题优先；无可用题型则不出） |
| 2 ≤ N ≤ 4 | 有连线题 | 1 道连线题 |
| 5 ≤ N ≤ 8 | 有连线题 | 2 道连线题 |
| 9 ≤ N ≤ 12 | 有连线题 | 3 道连线题 |
| N 任意 | 无连线题 | 过滤低层后有可用题型则每知识点各1道，否则不出 |

> **连线类题型**（`图音连线`、`图图连线`、`图音连线3-3`、`图图连线3-3`）可将多个知识点融合为一道题，因此道数按区间计算而非"每知识点1道"。

**兜底校验（路径 B）：**
- 接口返回的所有题目必须归属到某个有效层级（题型在允许范围内）
- 整组期望总题数 = 各层期望之和，受全局20道上限约束

---

#### 3.7 脚本内部函数说明

| 函数 | 说明 |
|------|------|
| `load_question_type_map()` | 读取题型编码↔中文名双向映射，返回 `(mapping, reverse_mapping)` |
| `load_requirement_rules()` | 读取需求对应关系，处理合并单元格和 `--`，返回 `(rules, level_groups)` |
| `load_db_mastery(appoint_id)` | 从数据库查询指定预约 ID 的知识点掌握度列表 |
| `load_api_questions(json_file)` | 解析 `json.json`，提取题目的题型、知识点、能力标签，跳过非题目模板 |
| `load_lesson_scope(lesson_type, level)` | 从 `GC_L0~L2_有效学习目标.xlsx` 读取指定课型+Level 的知识点类型+能力项组合 |
| `_calc_expected_lianxian(n)` | 计算 mastery=3 层连线题期望道数（N∈[2,4]→1, [5,8]→2, [9,12]→3） |
| `verify(...)` | 核心校验函数，按分组逐一校验并打印报告 |
